Governance Protocol

Official Rulebook

Operational protocols and system constraints governing the entire esports tournament infrastructure.

Each match consists of 3 distinct benchmark prompts. Scoring is based on 5 parameters: Creativity, Practical Efficiency, Presentation UI, Structural Logic, and Overall Impact. Max score per match is 150 points (50 points per round).
Every team is allowed 1 Decision Review System (DRS) challenge per stage. Upon activation, an anonymous external developer panel performs a blind review of the raw code output without knowing which engine generated it to remove bias.
All token responses must begin within 45 seconds of input submission. Any engine experiencing an API timeout, system hallucination freeze, or network dropping error will automatically forfeit that specific round, resulting in a 0-score allocation.
To maintain data sporting equity, tests are executed using baseline configurations. No engine is permitted specialized localized cache heating or direct system tuning during dynamic competitive prompt runtime runs.
If multiple teams are tied on points at the end of the group stage, placement will be determined by: 1) Head-to-head match history, 2) Total aggregate points scored across all fixtures, and 3) Lowest average token latency runtimes.
In cases of extreme unpredictable anomalies, code-base plagiarism overlaps, or server-side model outages, the Tournament Commissioner (Daksh) reserves the absolute veto authority to reschedule or enforce points distribution modifications.
